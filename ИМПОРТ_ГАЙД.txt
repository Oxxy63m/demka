================================================================================
  ШПАРГАЛКА: ИМПОРТ ДАННЫХ В БД ЧЕРЕЗ EXCEL / CSV
================================================================================

1. КУДА КЛАСТЬ ФАЙЛЫ
   Папка: resources (в корне проекта).
   Либо поменять путь в App/config.py → DATA_DIR.

2. КАКИЕ ФАЙЛЫ НУЖНЫ (имена точно так!)

   user_import.xlsx          — пользователи
   Tovar.xlsx                — товары
   Заказ_import.xlsx         — заказы
   Пункты выдачи_import.xlsx — адреса пунктов выдачи (один адрес в строке, первый столбец)
   order_items.xlsx          — позиции заказов (необязательно)

   Если есть только CSV:
   — Открыть в Excel → «Сохранить как» → тип «Книга Excel (*.xlsx)».
   — Или переименовать .csv в .xlsx не подойдёт — нужен именно Excel.

3. ЗАГОЛОВКИ СТОЛБЦОВ (первая строка — как в таблице)

   user_import.xlsx:
   Логин | Пароль | ФИО | Роль сотрудника
   Роль: администратор, менеджер, клиент (или по-русски).

   Tovar.xlsx:
   Артикул | Наименование товара | Цена | Единица измерения | Поставщик |
   Производитель | Категория товара | Действующая скидка | Кол-во на складе |
   Описание товара | Фото
   (Фото — имя файла, например picture.png; если пусто — подставится picture.png)

   Заказ_import.xlsx:
   Дата заказа | Дата доставки | ФИО авторизированного клиента | Номер клиента |
   Адрес пункта выдачи | Код для получения | Статус заказа
   Адрес пункта выдачи: число 1, 2, 3… = номер строки в «Пункты выдачи_import.xlsx»
   (строка 1 = первый адрес, строка 2 = второй и т.д.)

   Пункты выдачи_import.xlsx:
   Без заголовка. В первом столбце — адрес (одна строка = один пункт).

   order_items.xlsx:
   Номер заказа | Номер товара | Количество | Цена
   (Номер заказа и Номер товара — id из уже загруженных заказов и товаров.)

4. КАК ЗАПУСТИТЬ ИМПОРТ

   В папке проекта в терминале:
   python import_to_db.py

   Порядок загрузки скриптом: пункты выдачи → пользователи → товары → заказы → позиции заказов.

5. ЕСЛИ ЧТО-ТО НЕ ТАК

   — Ошибка «нет файла» → проверь папку resources и имена файлов (см. п. 2).
   — Ошибка БД → проверь App/config.py (хост, пароль, имя базы).
   — Заказы не находятся по клиенту → в заказах «ФИО авторизированного клиента»
     должно совпадать с «ФИО» в user_import.xlsx (регистр не важен).
   — Код скрипта для ручного набора или правок лежит в import_to_db.txt.
     Чтобы запускать, скопируй в import_to_db.py.
