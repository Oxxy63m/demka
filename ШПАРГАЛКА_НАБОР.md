# Шпаргалка: порядок набора кода на другой машине

Набирать файлы **в таком порядке** (сверху вниз). В каждом файле — **один и тот же приём**: импорты → константы/данные → функции/классы.

---

## 1. Папки

Создать папки: `App`, `ui`, `logic`, `resources`.

---

## 2. App/config.py

**Схема:** импорт os → ROOT, DATA_DIR, UI-словарь, DB_CONFIG → остальные константы (роли, статусы, PICKUP_POINTS).

**Что запомнить:** все пути через `os.path.join(ROOT, "ui", "login.ui")` и т.п. Пароль БД в `DB_CONFIG`.

---

## 3. main.py

**Схема:** импорты (PySide6, App.config, Login, Main) → константы шрифта → функция `app_stylesheet()` → функция `main()` (создать resources при необходимости → QApplication → иконка, стили, шрифт → цикл: Login → exec → Main → showMaximized → app.exec) → `if __name__ == "__main__": main()`.

**Что запомнить:** один раз `create_db.py`, потом всегда `main.py`. В main только запуск окна входа и главного окна.

---

## 4. create_db.py

**Схема:** импорты (os, sys, psycopg2, App.config) → DB_NAME = "demka" → функция main(): подключение к postgres → создать БД если нет → открыть DB.sql → подключение к demka → cursor.execute(schema_sql) → commit → закрыть. В конце `if __name__`: try main(), except print.

**Что запомнить:** два подключения: сначала к postgres (создать базу), потом к demka (выполнить DB.sql).

---

## 5. DB.sql

Набирать готовый скрипт таблиц. Порядок: справочники (roles, suppliers, …) → users → products → orders → order_items → INSERT начальных данных.

---

## 6. logic/auth.py

**Схема:** sys.path → импорт auth_user из App.db → функция login(login_text, password): return auth_user(...) → функция get_guest_user(): return {"full_name": "Гость", "role": "guest"}.

**Что запомнить:** auth только вызывает db и возвращает гостя.

---

## 7. App/Login.py

**Схема:** импорты (PySide6, loadUiType, config, logic.auth) → loadUiType(UI["login"]) → class Login(BaseLogin, Ui_Login): __init__ (setupUi, кнопки .clicked.connect) → _set_logo (если есть icon.png) → _on_login (взять логин/пароль → auth_login → accept) → _on_guest (get_guest_user → accept) → get_user (return self.user).

**Что запомнить:** окно входа = форма из .ui + две кнопки (Войти → проверка по БД, Гость → без проверки).

---

## 8. App/db.py

Самый большой файл. **Общий приём для каждой функции:**  
`conn = psycopg2.connect(**DB_CONFIG)` → `cur = conn.cursor(cursor_factory=RealDictCursor)` или `conn.cursor()` → `cur.execute(sql, params)` → `result = cur.fetchone()` или `fetchall()` → `cur.close(); conn.close()` → `return result`.

Порядок функций: auth_user → get_products_all, get_product_by_id → get_supplier_names, get_category_names, get_manufacturer_names → _get_or_create_* (supplier, manufacturer, category) → insert_product, update_product → product_in_orders, delete_product → заказы: _get_or_create_status_id, _get_or_create_pickup_point_id → get_orders_all, get_order_by_id → insert_order, update_order, delete_order → get_product_articles, build_order_article_string → get_or_create_role_id и *_conn для импорта → insert_order_items.

**Что запомнить:** везде connect → cursor → execute → fetch → close → return. Справочники: «найти id по имени, если нет — INSERT и вернуть id».

---

## 9. logic/product_list.py

Импорт из App.db (get_products_all, get_supplier_names, product_in_orders, delete_product) → четыре функции-обёртки: load_products, get_supplier_names, product_in_orders, delete_product — каждая просто вызывает одноимённую из db.

---

## 10. logic/order_list.py

Аналогично: импорт get_orders_all, delete_order из App.db → load_orders() и delete_order(order_id) — обёртки.

---

## 11. logic/order_edit.py и logic/product_edit.py

Та же идея: импорты из config и App.db → функции load, save, get_statuses, get_users_list / get_category_names, get_manufacturer_names — вызывают db и config.

---

## 12. App/Main.py, App/Orders.py, App/OrderForm.py, App/ProdForm.py, App/Card.py, App/Cart.py

**Общая схема окна:** loadUiType(UI["..."]) → class ИмяОкна(Base, Ui): __init__ (setupUi, привязка кнопок .clicked.connect к своим методам) → методы _on_что (взять данные с формы → вызвать logic или db → обновить экран или закрыть).

**Что запомнить:** у каждого окна — один .ui файл, setupUi, кнопки подключаются к методам, методы вызывают logic/db.

---

## 13. import_to_db.py

**Схема:** импорты (os, sys, pandas, psycopg2, config, db.get_or_create_*) → пути к xlsx (PATH_USERS и т.д.) → функции import_users, import_products, import_orders, import_order_items (в каждой: read_excel → цикл по строкам → get_or_create_* для справочников → cursor.execute INSERT) → main(): подключение к БД → вызов всех import_* → commit.

**Что запомнить:** для каждой таблицы — своя функция import_*. Справочники заполняются через get_or_create_* из db.

---

## 14. Файлы .ui

Их не набирают вручную — создают в Qt Designer и сохраняют в папку `ui/`. Если на другой машине нет .ui, их нужно скопировать или получить из проекта.

---

## Краткий чек-лист

1. Папки App, ui, logic, resources  
2. config.py — пути и DB_CONFIG  
3. main.py — точка входа  
4. create_db.py — создание БД  
5. DB.sql — таблицы  
6. logic: auth → product_list → order_list → order_edit → product_edit  
7. App: db.py (по шаблону connect→cursor→execute→fetch→close) → Login → Main → Orders, OrderForm, ProdForm, Card, Cart  
8. import_to_db.py — если нужен импорт из Excel  

**Паттерн для запоминания:** везде «импорты → данные/константы → функции». В db — «conn, cur, execute, fetch, close». В окнах — «setupUi, connect(кнопка), метод вызывает logic/db».

---

## Шаблон одной функции в db.py (повторять)

```python
def имя_функции(аргументы):
    conn = psycopg2.connect(**DB_CONFIG)
    cur = conn.cursor(cursor_factory=RealDictCursor)  # или conn.cursor()
    cur.execute("SELECT ...", (параметры,))
    res = cur.fetchone()  # или fetchall()
    cur.close()
    conn.close()
    return res
```

Для INSERT/UPDATE после execute делать `conn.commit()`. Справочники: сначала SELECT по имени, если нет — INSERT, потом снова SELECT и вернуть id.
