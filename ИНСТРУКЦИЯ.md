# Пошаговая инструкция (даже без опыта в Python)

**Если код нужно будет набирать вручную на другой машине** — используйте **ШПАРГАЛКА_НАБОР.md**: там порядок файлов и что в каждом набирать (паттерны, без лишнего).

## Что это за программа

Это приложение «Система учёта товаров»: каталог товаров, заказы, вход под разными ролями (гость, клиент, менеджер, администратор). Данные хранятся в базе PostgreSQL.

---

## Шаг 0. Что должно быть установлено

1. **Python 3** — с официального сайта python.org (при установке отметьте «Add Python to PATH»).
2. **PostgreSQL** — сервер базы данных (например, с postgresql.org или через установщик ОС).
3. В папке проекта в командной строке выполнить один раз:
   ```
   pip install -r requirements.txt
   ```

---

## Шаг 1. Создать базу данных (один раз)

В командной строке перейти в папку проекта и выполнить:

```
python create_db.py
```

**Что делает:** создаётся база с именем `demka` и все таблицы по скрипту `DB.sql`.  
Пароль к PostgreSQL и имя базы задаются в `App/config.py` (раздел «Подключение к PostgreSQL»).

---

## Шаг 2. Загрузить данные из Excel (если есть файлы)

Положить в папку с данными (по умолчанию — путь в `App/config.py`, переменная `DATA_DIR`) файлы:

- `user_import.xlsx` — пользователи  
- `Tovar.xlsx` — товары  
- `Заказ_import.xlsx` — заказы  
- при необходимости: `Пункты выдачи_import.xlsx`, `order_items.xlsx`

Затем выполнить:

```
python import_to_db.py
```

**Что делает:** читает Excel и заполняет таблицы в базе. Папку с файлами можно изменить в `App/config.py` (переменная `DATA_DIR`).

---

## Шаг 3. Запустить программу

В командной строке:
```
python main.py
```

Или просто дважды щёлкнуть по файлу **запуск_программы.bat** (Windows) — откроется то же самое.

**Что делает:** открывается окно входа. После входа — главное окно с каталогом товаров. Дальше всё через кнопки в интерфейсе.

---

## Кратко: порядок действий

| Порядок | Команда              | Когда использовать        |
|--------|----------------------|---------------------------|
| 1      | `python create_db.py`   | Один раз (создание БД)   |
| 2      | `python import_to_db.py`| Когда есть Excel для импорта |
| 3      | `python main.py`        | Каждый раз для работы с программой |

---

## Где что лежит (чтобы проще выучить)

| Файл / папка      | Назначение |
|-------------------|------------|
| `main.py`         | Запуск программы (точка входа). |
| `create_db.py`    | Создание базы и таблиц. Запускать первым. |
| `import_to_db.py` | Импорт из Excel в базу. Запускать после создания БД. |
| `DB.sql`          | Текст скрипта создания таблиц (выполняется через `create_db.py`). |
| `App/config.py`   | Настройки: путь к папке с Excel/фото, пароль БД, имена файлов интерфейса. Менять при смене компьютера или пути. |
| `App/`            | Окна и логика приложения (вход, каталог, заказы, корзина и т.д.). |
| `ui/`             | Файлы разметки окон (формат .ui). |
| `logic/`          | Функции логики: вход (auth), список товаров (product_list), заказы (order_list, order_edit), форма товара (product_edit). |
| `запуск_программы.bat` | Двойной щелчок — то же, что команда `python main.py`. |

---

## Если что-то не работает

- **Ошибка подключения к базе** — проверить, что запущен PostgreSQL и в `App/config.py` верные host, port, user, password, database.
- **Не находит Excel** — проверить путь `DATA_DIR` в `App/config.py` и что файлы лежат именно в этой папке.
- **Ошибка при импорте** — проверить имена листов и столбцов в Excel (должны совпадать с теми, что ожидает `import_to_db.py`).

После изменения `App/config.py` программу и импорт нужно запускать заново.
